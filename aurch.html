Aurch

What is aurch:
Aurch is a bash script used to automate building and optionally installing AUR packages. It's standout feature is building packages 'from within' a chroot, not to be confused with building package 'in' a chroot. This is a feature none of the currently popular AUR helpers have implemented.

Although very uncommon, it is not unheard of for AUR packages to contain intentionally malicious code, not to mention the more common unintentional errors that could potentially create issues.

Building packages from within a chroot mitigates any potential unwanted changes being made to the system during the build process. The build process when using aurch is performed isolated from the host, inside an nspawn container.

To be clear, an AUR package could be designed with malicious intent or otherwise unwanted features that could be implemented after installation quite easily. Aurch prevents it 'during the build process'.

Expecting code review by the user for packages is actually quite unrealistic. More realistic would be verifying the source code downloaded to build a package is actually from the original source provided by the development. Checksums implemented in the PKGBUILD automate that step. That's if the PKGBUILD is built with integrity and has not been tampered with.

I've made an effort with aurch to prevent a tiny percentage of a potential breach of trust in a huge chain of trust on my systems. I'd like to share my efforts with anyone interested in using it.

Intended user:
I made aurch to use on my personal systems. I cannot offer support or help other than passing on what I learn over time.

1) Anyone using an unvetted script from an unknown person, absolutely needs to be able to read the script to decipher what it's actually doing.

2) I'd advise anyone wanting to try or use aurch, to at least be able to troubleshoot and fix any package related issues on their systems without assistance. AUR packages can and will cause issues under the management of the best AUR helpers.

3) It'd be cool to see users with some shell scripting skills and understanding of Arch packages, to provide constructive feedback. I'd be glad to hear from you. Shoot me a message on the Arch reddit, u/Cody_Learner.

Why no AUR package:
See intended user. Mostly because I'm not able to, or do I want to support a potential large (or even small) number of users needing help. There's more, with this being the most prominent one I don't mind discussing.
